# Feedback

Patterns the user has either **corrected** ("don't do X") or **validated** ("yes, keep doing that"). The most behaviorally important page in this wiki — entries here should directly change how the assistant acts in future sessions.

> _Empty on purpose. Each entry should follow the structure below: rule, why, when to apply._

## Format for each entry

```markdown
### <short rule>

**Rule:** <the concrete behavior to do or avoid>
**Why:** <reason the user gave, often a past incident or strong preference>
**How to apply:** <where/when this kicks in>
**Source:** <raw/... file or [[log#YYYY-MM-DD]]>
```

## Corrections

_Things the user told the assistant to stop doing._

## Validated approaches

_Non-obvious approaches the user explicitly endorsed. Just as important as corrections — saving only corrections causes drift away from approaches already known to work._
